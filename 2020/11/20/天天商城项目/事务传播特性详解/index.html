事务传播特性详解

spring
发布日期:   2020-11-20
更新日期:   2020-12-26
文章字数:   1.8k

事务传播特性—-详解：

required：使用当前的事务，如果当前没有事务，则创建一个事务，子方法是必须运行在一个事务中的，如果当前存在事务，则加入这个事务，成为一个整体。
supports:如果当前有事务，则使用事务，如果当前没有事务，则不使用事务
mandatory：该传播属性强制必须存在一个事务，如果不存在，则抛出异常
required_new:如果当前有事务，则挂起该事务，并且自己创建一个新的事务给自己使用；如果当前没有事务，则同required
not_support:如果当前有事务，则把事务挂起，自己不使用事务去进行数据库操作
never:如果当前事务存在，则抛出异常
nested： 如果当前有事务，则开启子事务（嵌套事务），嵌套事务是独立提交或者回滚；如果当前没有事务，则同required
但是如果父事务提交，则会携带子事务一起提交。如果父事务回滚，则子事务会一起回滚。相反，子事务异常，则父事务可以不会滚（捕获子事务的异常），也可以回滚。
有异常才会回滚事务，如果异常被捕获，也就是try，catch，就不会回滚事务了，
发生异常，或者抛出异常都会回滚事务

准备工作

  1. 导入test依赖

    <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>

  2. 编写test类 

    
@RunWith(SpringRunner.class)
@SpringBootTest(classes = Application.class)
public class TransTest {
    @Autowired
    private TestTransService testTransService;
    /*
    * 事务传播特性
    *   required：
    *   supports：
    *   mandatory：
    *   required_new:
    *   not_supported:
    *   never:
    *   nested:
    * */
    @Test
    public void demo(){
        testTransService.saveParent();
        testTransService.saveChildren();
    }
}

  3. 编写需要的测试方法

    @Service
    public class TestTransService {
        @Autowired
        private StuMapper stuMapper;
        public void saveParent(){
            Stu stu=new Stu();
            stu.setName("parent");
            stu.setAge(55);
            stuMapper.insert(stu);
        }
        public void saveChildren(){
            saveChild1();
            int a=1/0;
            saveChild2();
        }
        public void saveChild1(){
            Stu stu1=new Stu();
            stu1.setName("child-1");
            stu1.setAge(11);
            stuMapper.insert(stu1);
        }
        public void saveChild2(){
            Stu stu2=new Stu();
            stu2.setName("child-2");
            stu2.setAge(22);
            stuMapper.insert(stu2);
        }

REQUIRED

required：使用当前的事务，如果当前没有事务，则创建一个事务，子方法是必须运行在一个事务中的，如果当前存在事务，则加入这个事务，成为一个整体。

(1)demo方法没有开启事务，直接测试
报错：java.lang.ArithmeticException: / by zero
parent，child1入库
结论：在报错之前，已经执行的代码会进行数据库操作，之后代码没有执行，也就没有操作数据库

(2)在demo()方法上开启事务propagation = Propagation.REQUIRED，进行测试
/*
* 事务传播特性
* required：
* supports：
* mandatory：
* required_new:
* not_supported:
* never:
* nested:
* */

`@Transactional(propagation = Propagation.REQUIRED)`

@Test
public void demo(){
    testTransService.saveParent();
    testTransService.saveChildren();

报错：java.lang.ArithmeticException: / by zero

结论：
demo()方法开启REQUIRED事务，事务具有传播特性，方体体里面的方法也具有事务，所以报错后数据库进行了回滚操作

如果demo方法没有开启事务，在saveChildren()上开启了事务（ @Transactional(propagation = Propagation.REQUIRED)），则会保存saveParent()方法插入的数据。

supports

supports:如果当前有事务，则使用事务，如果当前没有事务，则不使用事务

  1. demo方法上没有加事务，但是 saveChildren加SUPPORTS事务
    `@Transactional(propagation = Propagation.SUPPORTS)`
public void saveChildren(){
            saveChild1();
            int a=1/0;
            saveChild2();
        }

结论： 数据库新增parent,child-1数据
是因为外层demo没有事务，saveChildren也就跟着外层函数来，不增加事务。
2. demo方法上加Required事务，但是 saveChildren加SUPPORTS事务

结论: 没有数据入库，事务就进行了回滚操作

mandatory

该传播属性强制必须存在一个事务，如果不存在，则抛出异常

  1. 把调用方demo的Required事务给注释掉，在被调用方saveChildren使用事务mandatory
    会抛出没有事务的异常
  2. demo方法上加Required事务，saveChildren加mandatory事务

required_new

required_new:如果当前有事务，则挂起该事务，并且自己创建一个新的事务给自己使用；如果当前没有事务，则同required

  1. 把调用方demo的Required事务给注释掉，在被调用方saveChildren使用事务required_new
    会有一条parent记录，saveChildren自己本身有事务，进行了回滚
  2. demo方法上加Required事务，saveChildren加required_new事务
    此时有两个事务，
    数据库此时并没有任何数据
    这是因为saveChildren回滚，本身有/0异常，会影响到demo方法，也会回滚
    由于抛出空指针异常，所以saveChildren()这个新事物会回滚，而抛出的异常会传递给父方法，父方法的事务也会回滚，所有没有一条记录
  3. demo方法上加Required事务，saveChildren加required_new事务
    把saveChildren的/0注释掉，放到demo方法中的时候，
    数据库有child1 ,chird2
    结论：可以证明required_new创建的时新事务

not_support

如果当前有事务，则把事务挂起，自己不使用事务去进行数据库操作

  1. 把调用方demo的Required事务给注释掉，在被调用方saveChildren（有异常）使用事务not_support
    parent，child1入库,没有回滚
  2. 调用方demo使用Required事务，在被调用方saveChildren（有异常）使用事务not_support
    child1入库

never

如果当前事务存在，则抛出异常
1.调用方demo使用Required事务，在被调用方saveChildren（有异常）使用事务never
会抛出另一个异常非法事务状态
2.调用方demo不使用Required事务，在被调用方saveChildren（有异常）使用事务never
会抛出异常 /0 parent，child1入库

nested

如果当前有事务，则开启子事务（嵌套事务），嵌套事务是独立提交或者回滚；如果当前没有事务，则同required

1.调用方demo使用Required事务（有异常），在被调用方saveChildren（没有异常）使用事务nested
数据库没有数据
如果在外层方法发生异常，被调用方法也会回滚
父事务回滚，则子事务会一起回滚

**区别与required_new**  主方法发生异常，子方法不会收到影响 新开启一个新的事务  看前面

2.调用方demo使用Required事务（没有异常），在被调用方saveChildren（有异常）使用事务nested
会抛出异常 /0 没有数据库入库

在外层方法demo追加try catch，主方法其他操作不会受到子事务的影响

@Transactional(propagation = Propagation.REQUIRED)
public void demo(){
    testTransService.saveParent();
    try {
        //相当于数据库中的savepoint 捕获异常之后，不影响其他事务操作
        testTransService.saveChildren();
    } catch (Exception e) {
        e.printStackTrace();
    }
}
但是如果父事务提交，则会携带子事务一起提交。如果父事务回滚，则子事务会一起回滚。 相反，子事务异常，则父事务可以不回滚（捕获子事务的异常），也可以回滚。 有异常才会回滚事务，如果异常被捕获，也就是try，catch，就不会回滚事务了， 发生异常，或者抛出异常都会回滚事务
